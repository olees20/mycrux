# Media handling and retention

All uploads use private, purpose-specific buckets. Server actions validate the declared MIME type, filename extension, file signature, decoded image dimensions and pixel count, then re-encode images to WebP without source metadata. A 480-pixel thumbnail is stored beside each image. MP4 ascent clips are signature-checked and remain unmodified; users should remove sensitive metadata before upload.

Paths are generated by the server as `<gym UUID>/[owner UUID/]<asset UUID>.<extension>`. Database triggers reject other shapes, cross-tenant paths, thumbnail uploads without a source object and uploads over the gym plan quota. Signed URLs are short-lived and RLS checks the requesting user's gym access.

Deleting product content should mark its `media_assets` row deleted and set a retention review date. Storage removal is asynchronous so accidental deletion can be recovered; reported or legally retained media is quarantined instead. Operational cleanup must remove both the source and thumbnail only after the retention date and after confirming there is no open abuse, safety, fraud or legal hold. Audit and waiver evidence must never be inferred from media deletion.
